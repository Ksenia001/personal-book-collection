# 2 Варианты использования

<a name="view_book_collection"/>

## 2.1 Просмотр коллекции книг

**Описание.** Вариант использования "Просмотр коллекции книг" позволяет пользователю увидеть все книги, добавленные в его библиотеку.  
**Предусловия.** Приложение открыто.  
**Основной поток.**
1.  Система получает список всех книг из коллекции.
2.  Система отображает книги в виде списка или карточек на главном экране. Если коллекция пуста, выполняется альтернативный поток А1.
3.  Вариант использования завершается.

**Альтернативный поток А1 (Коллекция пуста).**
1.  Система отображает на главном экране сообщение "Ваша библиотека пока пуста" и предложение добавить первую книгу.
2.  Вариант использования завершается.

<a name="add_book"/>

## 2.2 Добавление книги

**Описание.** Вариант использования "Добавление книги" позволяет пользователю добавить новую книгу в свою коллекцию.  
**Предусловия.** Пользователь находится на главном экране приложения.  
**Основной поток.**
1.  Пользователь нажимает кнопку "+ Добавить книгу".
2.  Система отображает диалоговое окно "Информация о книге" с пустыми полями для ввода.
3.  Пользователь вводит данные о книге. Обязательными для заполнения являются поля "Название" и "Автор".
4.  Пользователь нажимает кнопку "Сохранить".
5.  Система проверяет, что все обязательные поля заполнены. Если нет, выполняется альтернативный поток А2.
6.  Система создает новую запись о книге и добавляет ее в коллекцию.
7.  Система закрывает диалоговое окно.
8.  Вариант использования завершается.

**Альтернативный поток А2 (Не заполнены обязательные поля).**
1.  Система отображает сообщение об ошибке рядом с незаполненными обязательными полями.
2.  Диалоговое окно остается открытым.
3.  Возврат к п.3 основного потока.

**Дополнительная информация.** Пользователь может отменить действие на любом шаге до сохранения. В этом случае выполняется альтернативный поток А3.

**Альтернативный поток А3 (Отмена действия).**
1.  Пользователь нажимает кнопку "Отмена".
2.  Система закрывает диалоговое окно без сохранения данных.
3.  Вариант использования завершается досрочно.

**Постусловия.** Выполняется вариант использования "[Просмотр коллекции книг](#view_book_collection)".

## 2.3 Редактирование книги

**Описание.** Вариант использования "Редактирование книги" позволяет пользователю изменить информацию о ранее добавленной книге.  
**Предусловия.** В коллекции есть хотя бы одна книга. Пользователь находится на главном экране.  
**Основной поток.**
1.  Пользователь нажимает кнопку "Редактировать" у выбранной книги в списке.
2.  Система отображает диалоговое окно "Информация о книге", заполненное текущими данными этой книги.
3.  Пользователь изменяет данные в полях.
4.  Пользователь нажимает кнопку "Сохранить".
5.  Система проверяет, что обязательные поля "Название" и "Автор" не остались пустыми. Если это так, выполняется альтернативный поток А2 из сценария "[Добавление книги](#add_book)".
6.  Система сохраняет обновленную информацию о книге.
7.  Система закрывает диалоговое окно.
8.  Вариант использования завершается.

**Дополнительная информация.** Пользователь может отменить действие. В этом случае выполняется альтернативный поток А3 из сценария "[Добавление книги](#add_book)".

**Постусловия.** Выполняется вариант использования "[Просмотр коллекции книг](#view_book_collection)".

<a name="delete_book"/>

## 2.4 Удаление книги

**Описание.** Вариант использования "Удаление книги" позволяет пользователю удалить книгу из своей коллекции.  
**Предусловия.** В коллекции есть хотя бы одна книга. Пользователь находится на главном экране.  
**Основной поток.**
1.  Пользователь нажимает кнопку "Удалить" у выбранной книги в списке.
2.  Система отображает диалоговое окно с запросом подтверждения: "Вы уверены, что хотите удалить книгу «Название книги»?".
3.  Пользователь нажимает кнопку "Да, удалить".
4.  Система удаляет запись о книге из коллекции.
5.  Система закрывает диалоговое окно подтверждения.
6.  Вариант использования завершается.

**Альтернативный поток А4 (Отмена удаления).**
1.  На шаге 3 основного потока пользователь нажимает кнопку "Отмена".
2.  Система закрывает диалоговое окно подтверждения без удаления книги.
3.  Вариант использования завершается досрочно.

**Постусловия.** Выполняется вариант использования "[Просмотр коллекции книг](#view_book_collection)".

<a name="search_book"/>

## 2.5 Поиск книги

**Описание.** Вариант использования "Поиск книги" позволяет пользователю отфильтровать список книг по названию или автору.  
**Предусловия.** Пользователь находится на главном экране.  
**Основной поток.**
1.  Пользователь вводит текст в поле "Поиск".
2.  Система в реальном времени фильтрует список книг на главном экране, отображая только те, у которых название или автор содержат введенный текст. Если совпадений не найдено, выполняется альтернативный поток А5.
3.  Вариант использования завершается, когда пользователь очищает поле поиска.

**Альтернативный поток А5 (Книги не найдены).**
1.  Система отображает сообщение "По вашему запросу ничего не найдено".
2.  Список книг на экране пуст.
3.  Возврат к п.1 основного потока (пользователь может изменить запрос).
