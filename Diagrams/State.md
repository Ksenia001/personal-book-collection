# Диаграмма состояний (Схема экранов)

Данная диаграмма, также известная как Wireflow, визуализирует жизненный цикл пользовательского интерфейса (GUI). Она показывает основные экраны приложения в виде мокапов и иллюстрирует, как действия пользователя (нажатия кнопок) вызывают переходы между этими экранами.

![Схема экранов GUI](https://github.com/Ksenia001/personal-book-collection/blob/main/Diagrams/Images/State.png?raw=true)

# Описание состояний (экранов)

| Экран (Состояние) | Описание |
|:---|:---|
| **Просмотр коллекции** | Основное состояние приложения. Пользователь видит список своих книг и имеет доступ ко всем основным функциям: добавлению, редактированию, удалению и поиску. |
| **Окно добавления книги** | Модальное состояние, в котором основной интерфейс заблокирован. Приложение ожидает ввода данных о новой книге от пользователя. |
| **Окно редактирования книги** | Модальное состояние, аналогичное добавлению, но с полями, предварительно заполненными данными существующей книги. |
| **Окно подтверждения удаления** | Модальное состояние, которое требует от пользователя подтвердить действие перед окончательным удалением книги из коллекции. |
